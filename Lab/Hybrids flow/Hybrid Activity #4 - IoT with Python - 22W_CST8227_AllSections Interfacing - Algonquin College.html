Hybrid Activity #4 - IoT with Python

Hide Assignment Information

Purpose

Welcome to your last hybrid activity of the module Introducing IoT, featuring Node-RED, and the course. This hybrid activity is a redux of Lab 08 - Toggle LED Node-RED Flow, with two notable differences. The first change: replace Node-RED (based on Node.JS and JavaScript) with Python [Ref. 1], another popular language in IoT. And the second difference: your Python script will publish the state of the LED to the Cayenne MQTT server in the cloud, much like you did in Lab 11 - Teensy in the Cloud

Features of the hybrid activity include:

  • circuit layout: fritzing layout for a discrete LED (refer to Resources module on Brightspace)
  • hardware: Teensy, switch (i.e. pushbutton), discrete LED, tri-colour LED, thermistor resistor(s), wires, breadboard
  • communication: USB cable (Teensy); Internet (MQTT)
  • software: StandardFirmataPlus sketch (Teensy); Python on your BYOD

Due Dates

  • deliverables => upload & submit to Brightspace before Due Date

Before You Begin

Your Teensy must be running the Standard Firmata Plus (SFP+) sketch.

Remember to close / quit the Arudino IDE and Teensy loader after uploading the SFP+ sketch. 

Before You Begin #2

Download and install Python version 3.9.x for your BYOD.

Your Python environment should be very similar to mine:

Use Python's package manager, pip, to install:

  • pyfirmata
  • cayenne-mqtt

Notes:

  • other packages will be installed too, such as pyserial; in this case, pyFirmata depends on pyserial
  • you can safely ignore the warning to upgrade pip

Rubric

The high-level rubric for this activity:

A+ - replace discrete LED with a tri-colour LED (analog); add 4 slider widgets to the Cayenne dashboard, one each for Red, Green, Blue and Dimmer

B - replace discrete LED with a thermistor and display the temperature in Celsius, Fahrenheit and Kelvin on the Cayenne dashboard

C - toggle the onboard LED using Python. That is, repeat the steps written in this hybrid activity for a 'C'

D - missing one deliverable; solution does not solve problem

F - no deliverables

Deliverables xor Demonstration

Do one or the other but not both: deliverables xor demo.
